# Android SDK Dev Container Feature

Installs the Android SDK command-line tools, platform tools, a selected Android platform, and the matching build tools. It also installs the required Java version.

## Usage

Add the feature to `devcontainer.json`:

```jsonc
{
    "image": "mcr.microsoft.com/devcontainers/base:trixie",
    "features": {
        "ghcr.io/hafzaal/devcontainer-features/android-sdk:1.0.6": {}
    }
}
```

## Current Option Defaults

| Option | Default | Description |
| --- | --- | --- |
| `android_home` | `/usr/local/lib/android` | Directory in which to install the Android SDK. |
| `sdk_version` | `36` | Android platform version to install. |
| `build_tools_version` | `36.0.0` | Android build tools version to install. |
| `java_version` | `25` | OpenJDK version to install. |

For example:

```jsonc
{
    "features": {
        "ghcr.io/hafzaal/devcontainer-features/android-sdk:1.0.3": {
            "android_home": "/opt/android-sdk",
            "sdk_version": "34",
            "build_tools_version": "34.0.0",
            "java_version": "25"
        }
    }
}
```

## Environment

The feature configures these variables:

- `ANDROID_HOME=/usr/local/lib/android`
- `ANDROID_SDK_ROOT=/usr/local/lib/android`
- `PATH` includes `cmdline-tools/latest/bin` and `platform-tools`.

The installed tools include `adb` and `sdkmanager`.

## Testing

Run the SDK version test from the repository root:

```bash
devcontainer features test \
  --project-folder . \
  --features android-sdk \
  --filter specific_sdk \
  --skip-autogenerated \
  --skip-duplicated
```
